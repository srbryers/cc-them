# Sources: John Carmack

All sources are freely available public recordings, transcripts, posts, or archives. Source IDs (S1, S2, etc.) correspond to citations in profile.md.

## .plan Files (1995–2000)

- **[S1]** [John Carmack .plan Archive (GitHub)](https://github.com/ESWAT/john-carmack-plan-archive) — Mirror of Carmack's daily work logs from 1996–2010, originally shared via the finger protocol. Documents game engine development decisions, optimization strategies, and technical philosophy at id Software.
- **[S2]** [John Carmack .plan Archive — Readable Markdown](https://github.com/oliverbenns/john-carmack-plan) — Reformatted archive of Carmack's .plan files covering development of Quake, Quake II, Quake III Arena, and Doom 3.

## Talks & Conferences

- **[S3]** [QuakeCon Keynotes Archive (Internet Archive)](https://archive.org/details/john-carmack-quakecon-keynotes) — Collection of Carmack's annual QuakeCon keynote talks. The 2012 keynote covers software engineering as a social science, static analysis, code quality, and developer productivity. The 2013 keynote covers functional programming, shipping timelines, and code organization.
- **[S4]** [John Carmack on Software Patents (1997)](https://aaronbonner.io/post/15318032968/john-carmack-on-software-patents-1997) — Carmack's statement opposing software patents, describing them as barriers to independent invention.

## Interviews & Podcasts

- **[S5]** [Lex Fridman Podcast #309 — John Carmack: Doom, Quake, VR, AGI, Programming, Video Games, and Rockets (2022)](https://lexfridman.com/john-carmack/) — Over 5 hours covering programming languages, work habits, deep focus, game engine history, VR, AGI, and engineering philosophy.
- **[S6]** [Joe Rogan Experience #1342 — John Carmack (2019)](https://podcastnotes.org/joe-rogan-experience/carmack/) — Discussion of VR vs. AR, optimization philosophy, Armadillo Aerospace iterative testing, open source, AGI predictions, and engineering as value creation. Podcast notes and summary.

## Technical Writing & Essays

- **[S7]** [In-Depth: Static Code Analysis — John Carmack (Gamasutra/Game Developer, 2011)](https://www.gamedeveloper.com/programming/in-depth-static-code-analysis) — Detailed essay on integrating static analysis tools (Coverity, /analyze, PVS-Studio, PC-Lint) into game development workflows. Covers the mindset shift toward accepting that all code contains errors.
- **[S8]** [In-Depth: Functional Programming in C++ — John Carmack (Gamasutra/Game Developer, 2012)](https://www.gamedeveloper.com/programming/in-depth-functional-programming-in-c-) — Essay advocating pure functions, immutability, and functional patterns within C++ for game engines. Covers thread safety, testability, and the tension between OOP and functional approaches.
- **[S12]** [John Carmack on Inlined Code (2007 email, archived 2014)](https://cbarrete.com/carmack.html) — Email to a programmer mailing list covering inlined code, pure functions, execution awareness, the costs of abstraction, and a 2014 addendum on how functional programming addresses the root problems more directly. Also mirrored at [number-none.com](http://number-none.com/blow/john_carmack_on_inlined_code.html).

## Books & Biographical Sources

- **[S9]** [The Almanack of John Carmack — Artur Piszek (2024)](https://piszek.com/2024/06/25/the-almanack-of-john-carmack/) — Comprehensive compilation of Carmack's publicly stated principles on productivity, engineering philosophy, focus, adversarial thinking, and innovation, drawn from talks, interviews, and social media.
- [Masters of Doom — David Kushner (2003)](https://www.goodreads.com/book/show/222146.Masters_of_Doom) — Biography of id Software covering Carmack's engineering approach, shareware distribution philosophy, and development of Doom and Quake. Referenced for biographical context.

## AI & AGI

- **[S10]** [Exclusive Q&A: John Carmack's 'Different Path' to Artificial General Intelligence — Dallas Innovates (2023)](https://dallasinnovates.com/exclusive-qa-john-carmacks-different-path-to-artificial-general-intelligence/) — Interview covering Carmack's AGI probability estimates, his reasoning for pursuing AGI independently of major labs, and his belief that AGI source code will be surprisingly compact.
- **[S14]** [AGI and Robotics — John Carmack at Upper Bound 2025 (Amii)](https://www.amii.ca/videos/keen-technologies-research-directions-john-carmack-upper-bound-2025) — Talk on Keen Technologies research directions, sim-to-real robotics approaches, and grounding AI in physical reality.

## VR & Latency

- **[S16]** [Latency Mitigation Strategies — John Carmack (2013, archived by Dan Luu)](https://danluu.com/latency-mitigation/) — Technical paper on VR latency reduction techniques including GPU synchronization, late frame scheduling, view bypass, timewarp, and continuous timewarp. Establishes the 20ms perceptual threshold and strategies for achieving sub-3ms latency.

## Code Reviews & Source Analysis

- **[S15]** [Doom 3 Source Code Review: Interviews — Fabien Sanglard (2012)](https://fabiensanglard.net/doom3/interviews.php) — Interview with Carmack about Doom 3's C++ architecture, text-based file format decisions, and multi-processor rendering design. Part of Sanglard's comprehensive source code analysis.
- [Doom Source Code (GPL) — id Software (GitHub)](https://github.com/id-Software/DOOM) — The original Doom source code released December 23, 1997, re-licensed under GPL on October 3, 1999.
- [Doom 3 Source Code Review: Introduction — Fabien Sanglard](https://fabiensanglard.net/doom3/) — Detailed technical analysis of the id Tech 4 engine codebase.
